# Gate-OS v1.0.0-beta Release Notes

**Release date:** 2026-03-05  
**Codename:** Iron Gate  
**Status:** Public Beta — seeking community feedback

---

## Overview

Gate-OS v1.0.0-beta is the first public beta of our Ubuntu 24.04 LTS-based multi-environment operating system. This release delivers a complete, integrated platform for users who need to switch seamlessly between gaming, development, design, media production, and security research contexts — without rebooting.

**Switch latency target:** < 3 seconds  
**Supported architectures:** x86_64, ARM64

---

## What's New Since Alpha (v0.5.0)

- OTA update check mechanism (`gateos check-update`)
- All environments hardened with AppArmor + OCI seccomp profiles
- Prometheus metrics endpoint (`/metrics`) for monitoring dashboards
- Real-time WebSocket status stream (`/ws/status`) for mobile companion
- Ed25519 manifest signing (`gateos sign` / `gateos verify`)
- Ubuntu 24.04 LTS ISO builder with automated CI pipeline

## Installation

### Option A — Live ISO (Recommended)

1. Download the ISO from the Releases page.
2. Flash to USB with Ventoy or:
   ```bash
   sudo dd if=gate-os-1.0.0-beta.iso of=/dev/sdX bs=4M status=progress
   ```
3. Boot, select "Install Gate-OS".
4. Follow the on-screen installer.

### Option B — Overlay on Existing Ubuntu 24.04

See [docs/installation/guide.md](../installation/guide.md) for detailed instructions.

## Breaking Changes from v0.0.x Alpha

- API token is now **required** for `POST /switch`. Generate one with `gateos gen-token`.
- Signature format changed from 0.0.1 YAML schema to 1.0 JSON schema. Re-validate manifests.
- `gateos-api.service` now runs as `gateos` user (non-root). Adjust file permissions accordingly.

## Known Limitations (Beta)

| Limitation | Workaround |
|------------|-----------|
| OTA update `apply_update` is a stub — does not auto-apply | Run `sudo dpkg -i /var/cache/gateos/updates/gateos-*.deb` manually |
| Flutter companion app is scaffolded only | Use REST API directly: `POST /switch/{env}` |
| Switch latency can exceed 3 s if Podman image pull is needed | Pre-pull images with `podman pull gateos-{env}:latest` |
| AppArmor profiles are not auto-loaded on fresh Ubuntu install | Run `sudo bash scripts/enable-apparmor.sh` post-install |
| Multi-monitor GPU setup may require manual XRandR config in gaming env | Set `GATEOS_DISPLAY_POLICY=multi` in `/etc/gateos/gateos.conf` |

## Migration Guide (v0.0.6 → v1.0.0-beta)

1. **API token**: Generate and set a new token:
   ```bash
   gateos gen-token > /etc/gateos/api_token
   export GATEOS_API_TOKEN=$(cat /etc/gateos/api_token)
   ```

2. **Re-sign manifests**: If you have custom environment manifests:
   ```bash
   gateos gen-keypair --key-dir /etc/gateos/keys
   gateos sign manifests/my-custom-env.yaml
   ```

3. **Restart the API service**:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl restart gateos-api.service
   ```
